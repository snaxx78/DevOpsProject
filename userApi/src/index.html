CRUD Operations

Here are listed all the methods and API path we created. For a detailed documentation please check our complete doc: /api-docs

Method Path Parameter Infos
POST /user none Add a new user. Body format (json): {"username": "", "firstname": "", "lastname": ""}
PATCH /user none Update the content of an existing user. Body format (json): {"username": "", "firstname": "", "lastname": ""}
DELETE /user/:username username Delete a specific user.
GET /user/keys none Retrieve all keys.
GET /user/:username username Retrieve a specific user.

Other Routes

Here are the secondary API routes we defined for our project:

Method Path Parameter Infos
GET /health/api none Check the application health status
GET /health/redis none Check the Redis database health status
GET /api-docs none Obtain complete documentation of the API

Important: if you visit this page on the Azure WebApp deployment, for the sake of security no Redis DB has been connected to the API. This means that all Redis related functionalities are not available in the Azure deployment.
